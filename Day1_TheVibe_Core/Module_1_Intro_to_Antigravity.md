# Module 1: Intro to Antigravity IDE & Prompting

**Time:** 9:00 AM - 11:00 AM  
**Module Objective:** Introduce students to AI-assisted coding, how to think like a technical director, and how to write effective prompts.

---

## 1. Introduction to AI-Assisted Development (30 mins)
*Goal: Shift mindsets from writing boilerplate code to architecting solutions.*

- **What is AI-assisted development?**
    - The evolution from syntax-heavy programming to semantic, intention-based development.
    - Acting as a "Director" or "Architect" instead of just a typist.
- **The Antigravity IDE:**
    - Overview of the workspace: The chat interface, active file context, and artifact generation.
    - How the AI reads your file system and context to provide accurate code.
    - *Instructor Demo:* Show opening a blank folder and letting the AI set up an initial `index.html`.

## 2. Natural Language Prompting Basics (45 mins)
*Goal: Understand the structure of a successful prompt.*

- **What is a Prompt?** The basic unit of communication and instruction. It replaces boilerplate typing.
- **Good vs. Bad Prompts:**
    - ❌ *Bad:* "make a website for a coffee shop"
    - ✅ *Good:* "Create a responsive landing page for a coffee shop using HTML5 and Tailwind CSS. Include a dark-mode hero section with a call-to-action button, a CSS grid displaying 3 featured drinks, and a standard footer."
- **Context is King:**
    - The more specific you are about layout (Flexbox/Grid), colors (dark mode, specific hex codes), and structure (Semantic tags), the more professional the generated code.

## 3. Prompt Engineering for UI (45 mins)
*Goal: Advanced prompt structures and iterative refinement.*

- **Role-playing / Persona injection:** Instruct the AI to act as a "senior frontend developer specializing in modern UI/UX."
- **Iterative Refinement:**
    - Prompting is a conversation, not a single command.
    - Example workflow: 
        1. Generate the base layout.
        2. Prompt: "Make the navigation bar sticky with a glassmorphism blur."
        3. Prompt: "Ensure the feature cards stack horizontally on mobile devices."
- **Activity:** 
    - Have students write down their "dream" prompt for a portfolio site on paper/text editor.
    - Group discussion on how to improve and clarify those prompts before running them.
