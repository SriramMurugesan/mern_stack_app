# Instructor Demo Prompts - Day 1 Mini Project

This document contains the exact sequence of prompts you can copy and paste during the live classroom demo to generate the `mini_project/index.html` landing page step-by-step. This incremental approach teaches students the iteration process, rather than generating the whole page at once.

---

### Step 1: The Initial Foundation & Header
*Objective:* Show students how to set up the environment and semantic markers.

**Prompt to Copy:**
> "Act as an expert frontend developer. Create a single `index.html` file establishing a modern, responsive landing page for a creative agency called 'TheVibe'. Use strict Semantic HTML5 tags (`<header>`, `<main>`, `<section>`, `<footer>`). Import Tailwind CSS via a CDN script tag in the head. Apply the 'Inter' font from Google Fonts. For now, just generate the basic document structure and a fixed `<nav>` bar with the logo and desktop links."

---

### Step 2: Generating the Hero Section
*Objective:* Demonstrate layout control and layered CSS effects via natural language.

**Prompt to Copy:**
> "Perfect. Now, add a Hero section inside the `<main>` tag. It needs to be full height (`h-screen`) using Flexbox to center the content. Give it a dark indigo background. I want a large, bold headline that says 'Design at the speed of thought.', a descriptive subheadline, and two call-to-action buttons styled as rounded pills. To make it visually stunning, add an unsplash background image of a team collaborating, but apply a semi-transparent black overlay so the text remains completely legible."

---

### Step 3: Generating the Grid Section
*Objective:* Show the power of modern layout generation (Grid) for responsive content.

**Prompt to Copy:**
> "Below the Hero section, let's create a new `<section>` for our 'Core Principles'. Use a white background and give the section generous vertical padding. Inside, create a CSS Grid layout with 3 columns for desktop, but stack as 1 column on mobile. Create three separate feature cards using a light gray background, rounded corners, and a subtle drop shadow. The cards should be: 1. Semantic HTML5, 2. CSS Flexbox, 3. AI Prompting. Give each card a standout numbered badge at the top."

---

### Step 4: Iterative Refinement (Adding Polish)
*Objective:* Teach students that the AI is rarely perfect on the first try and demonstrate how to give direct feedback to fix or enhance elements.

**Prompt to Copy:**
> "The design structure is great, but the cards feel a bit static. Can you add a Tailwind hover effect to the 3 feature cards so that when the user mouses over them, they smoothly lift up (`-translate-y-2`) and the drop shadow becomes thicker? Also, add a similar hover scale up effect to the call-to-action buttons in the Hero section."

---

### Step 5: The Footer
*Objective:* Complete the semantic structure.

**Prompt to Copy:**
> "Finally, let's finish the page structure. Add a `<footer>` tag at the bottom with a very dark gray (almost black) background. Create a simple container that centers the text, displays 'TheVibe.' in a bold, large font, includes a standard copyright symbol for 2026, and lists neatly spaced out links for Privacy Policy, Terms of Service, and Contact Us. Ensure the text color is muted gray but turns white when hovered."
